<section id="s2">
<h2>Three Ways to Describe a Shape</h2>
<p>Every CAD kernel must answer one question: <em>how do you store a 3D solid in memory?</em> There are three fundamentally different answers. Each has trade-offs that determine what your software can and can't do.</p>

<div class="paradigm-grid">
  <div class="paradigm-card">
    <span class="paradigm-tag tag-brep">B-REP</span>
    <h3>Boundary Representation</h3>
    <p>A solid is defined by its boundary: <em>faces</em> (surface patches), <em>edges</em> (curves where faces meet), and <em>vertices</em> (points where edges meet). The topology (connectivity) is stored separately from the geometry (mathematical surface definitions).</p>
    <p style="margin-top:0.5rem"><strong>This is what your STEP files contain.</strong> Parasolid, ACIS, CGM, OCCT, Granite — all B-rep kernels. Every professional CAD system uses B-rep as its primary representation.</p>
    <p style="margin-top:0.5rem">Strengths: exact edge definitions, precise dimensions, clean toolpaths, lossless STEP export. Weakness: Booleans are hard (surface-surface intersection is where most bugs live).</p>
  </div>
  <div class="paradigm-card">
    <span class="paradigm-tag tag-nurbs">NURBS</span>
    <h3>Non-Uniform Rational B-Splines</h3>
    <p>The mathematical language B-rep uses to define its curves and surfaces. Control points don't lie on the curve — they <em>attract</em> it. Weights control how strongly each point pulls. This is how a kernel represents a perfect circle, a fillet, or a freeform surface.</p>
    <p style="margin-top:0.5rem"><strong>This is why fillets are smooth.</strong> Your end mill follows the evaluated NURBS curve — not a polygon approximation. The kernel computes the exact surface normal at every cutter-contact point.</p>
    <p style="margin-top:0.5rem">Key property: local control. Moving one control point only affects the nearby region. This is why parametric edits are fast.</p>
  </div>
</div>

<div class="paradigm-card" style="margin:1.5rem 0;border-color:var(--rule-bright)">
  <h3>NURBS Are Math, Not a File Type</h3>
  <p><strong>NURBS only describe curves and surfaces.</strong> They have no concept of "inside" or "outside." A NURBS surface is just a sheet floating in space — it's not a solid. B-rep adds solidity by stitching NURBS surface patches together into a closed shell with oriented normals (outward = outside).</p>
  <p style="margin-top:0.5rem">So when you hear "B-rep kernel" — the faces of that B-rep are NURBS surfaces. The edges are NURBS curves. NURBS is the <em>language</em>; B-rep is the <em>structure</em>.</p>
  <p style="margin-top:0.5rem;font-size:0.8rem;color:var(--text-secondary)"><strong>File types that carry NURBS:</strong> <code>.3dm</code> (Rhino) stores NURBS directly — can be loose surfaces, no solidity required. <code>.STEP</code> / <code>.x_t</code> / <code>.SAT</code> carry B-rep which contains NURBS as the surface/curve definitions. <code>.IGES</code> can carry either loose NURBS or B-rep solids — this ambiguity is why IGES imports are messy.</p>
</div>

<h3 style="margin-top:2rem">The Actual Math</h3>
<div class="paradigm-card" style="margin:1rem 0">
  <p style="font-size:0.8rem;color:var(--text-secondary)">A NURBS curve is defined by:</p>
  <div style="background:var(--bg-raised);padding:1rem;border-radius:3px;margin:0.75rem 0;font-family:var(--font-body);font-size:0.8rem;line-height:1.9;overflow-x:auto">
    <div><strong style="color:var(--accent)">C(u)</strong> = <strong>Σ</strong> N<sub>i,p</sub>(u) · w<sub>i</sub> · P<sub>i</sub> &nbsp;/&nbsp; <strong>Σ</strong> N<sub>i,p</sub>(u) · w<sub>i</sub></div>
    <div style="margin-top:0.75rem;color:var(--text-dim);font-size:0.7rem">
      P<sub>i</sub> = control points (the handles you drag in CAD)<br>
      w<sub>i</sub> = weights (how strongly each point pulls — this is the "Rational" in NURBS)<br>
      N<sub>i,p</sub> = B-spline basis functions of degree p (typically p=3, cubic)<br>
      u = parameter (0→1 walks along the curve)
    </div>
  </div>
  <p style="font-size:0.8rem;color:var(--text-secondary)"><strong>Why "Non-Uniform"?</strong> The knot vector controls where each basis function is active. Non-uniform spacing lets you pack more detail in one region without affecting others. Uniform = evenly spaced = less control.</p>
  <p style="margin-top:0.5rem;font-size:0.8rem;color:var(--text-secondary)"><strong>Why "Rational"?</strong> The weights w<sub>i</sub> are the key. With all weights equal (w=1), you get a polynomial B-spline — it can represent freeform shapes but <em>cannot exactly represent a circle</em>. Add weights, and you can represent any conic section exactly. This is why NURBS can describe a perfect fillet radius that your end mill follows precisely.</p>
  <p style="margin-top:0.5rem;font-size:0.8rem;color:var(--text-secondary)"><strong>Surfaces:</strong> A NURBS surface is the tensor product of two curves — one in u, one in v. The control points form a rectangular grid. This grid requirement is NURBS' biggest limitation.</p>
</div>

<div class="paradigm-card" style="margin:1.5rem 0;border-left:3px solid var(--accent)">
  <span class="paradigm-tag tag-nurbs">T-SPLINES</span>
  <h3>The NURBS Upgrade That Got Acquired</h3>
  <p>NURBS surfaces require a <em>rectangular grid</em> of control points. Want more detail on the nose of a car? You must add an entire row and column of control points across the whole surface — even areas that don't need it. This wastes data and makes editing painful.</p>
  <p style="margin-top:0.5rem"><strong>T-splines</strong> (Thomas Sederberg, BYU, ~2003) fix this. They allow <em>T-junctions</em> in the control mesh: you can add local detail without propagating control points globally. Think of it as "adaptive resolution" for surfaces.</p>
  <p style="margin-top:0.5rem"><strong>Autodesk acquired T-Splines Inc in 2011</strong>, killed the popular Rhino plugin (still a sore point in the Rhino community), and integrated the technology into Fusion 360. T-splines bridge the gap between SubD modeling (used in animation/games — organic, easy to sculpt) and NURBS (used in CAD — precise, manufacturable). In Fusion 360, you sculpt in T-spline mode, then convert to NURBS B-rep for manufacturing.</p>
  <p style="margin-top:0.5rem;font-size:0.8rem;color:var(--text-dim)">Sederberg's patents restricted wider adoption. Some began expiring ~2023. Whether open-source kernels will implement T-splines now is an open question.</p>
</div>

<div class="paradigm-card" style="margin:1.5rem 0">
  <span class="paradigm-tag tag-brep">SOLID MODELING</span>
  <h3>Solid Modeling vs Watertight B-rep</h3>
  <p>These are related but not the same thing.</p>
  <p style="margin-top:0.5rem"><strong>Watertight B-rep</strong> = a closed, manifold boundary. Every edge is shared by exactly two faces. No gaps, no T-junctions, no overlaps. The boundary completely encloses a volume. You can verify this with the <em>Euler-Poincaré formula</em>: <code>V − E + F = 2</code> for a simple solid (genus 0).</p>
  <p style="margin-top:0.5rem"><strong>Solid modeling</strong> = the full system for <em>creating, validating, and manipulating</em> solids. Booleans, filleting, shelling, chamfering, mass properties (volume, center of gravity, moments of inertia), interference detection. A solid modeling kernel <em>guarantees</em> that every operation either produces a valid solid or fails cleanly — no degenerate geometry slipping through.</p>
  <p style="margin-top:0.5rem">You CAN have a watertight B-rep without a solid modeler — manually stitch surfaces together in Rhino, close the boundary, done. But a solid modeler like Parasolid <em>guarantees</em> watertight results from every operation. That guarantee is what you're paying for when your CAD software licenses Parasolid. It's also why Booleans sometimes fail: the kernel refuses to produce an invalid result rather than giving you a broken solid.</p>
  <p style="margin-top:0.5rem;font-size:0.8rem;color:var(--text-secondary)">This matters for CAM: your toolpath generator assumes the model is a valid solid. If it isn't (gaps between faces, inconsistent normals), you get gouges, air cuts, or crashes. The kernel's job is to make sure that never happens.</p>
</div>

<div class="paradigm-card" style="margin:1.5rem 0">
  <span class="paradigm-tag tag-sdf">SDF / IMPLICIT</span>
  <h3>Signed Distance Fields</h3>
  <p>A fundamentally different approach. Every point in space has a number: its distance to the nearest surface. Negative = inside, positive = outside, zero = on the surface. The shape <em>is</em> the zero-level set of a mathematical function.</p>
  <p style="margin-top:0.5rem">CSG is trivially elegant: <code>union = min(a,b)</code>, <code>subtract = max(a,-b)</code>, <code>smooth blend = smin(a,b,radius)</code>. No topology headaches. No Boolean failures.</p>
  <p style="margin-top:0.5rem"><strong>The trade-off:</strong> no explicit edges or faces. Dimensioning to a specific edge is hard. Exporting to STEP requires meshing first — information is lost. This is why SDF excels at lattices and generative design but can't replace B-rep for precision machining. <em>Yet.</em></p>
</div>

<section id="s3">
<h2>Who Owns the Stack</h2>
<p>There are only a handful of geometry kernels in the world. A few companies control the mathematical foundations that <em>every</em> CAD and CAM application is built on.</p>

<h3 style="margin-top:2rem">Not All B-rep Kernels Are Equal</h3>
<p>Every major kernel stores faces, edges, and vertices in a half-edge data structure — but they diverge in <em>how</em> they handle the hard problems. The real differences:</p>

<div class="failure-grid" style="grid-template-columns:1fr">
  <div class="failure-card">
    <h3>Boolean Robustness</h3>
    <p>The #1 differentiator. When two NURBS surfaces intersect, the kernel must find the intersection curve — a notoriously fragile numerical computation. Each kernel has different tolerancing strategies, different fallback heuristics, different failure modes. This is why the same Boolean operation can succeed in SolidWorks and fail in Inventor. Same math problem, different solver.</p>
  </div>
  <div class="failure-card">
    <h3>Tolerance Philosophy</h3>
    <p><em>Parasolid</em> uses <strong>tolerant modeling</strong> — edges carry a tolerance band (typically ±0.001mm). If two surfaces don't meet exactly, the kernel approximates. This makes imports forgiving but can accumulate error. <em>ACIS</em> traditionally demands higher precision, using exact-match topology with separate healing tools for bad geometry. <em>CGM</em> uses a hybrid approach tuned for CATIA's aerospace workflows where tolerance stacking across large assemblies is the critical concern.</p>
  </div>
  <div class="failure-card">
    <h3>Architecture</h3>
    <p><em>ACIS</em> is modular — you license individual "husks" (components): modeling, booleans, blending, meshing. Your application only loads what it needs. <em>Parasolid</em> is monolithic — one library, everything included. More tightly integrated, harder to extend. <em>OCCT</em> is the most granular — 15+ packages you can mix and match.</p>
  </div>
  <div class="failure-card">
    <h3>Non-Manifold Support</h3>
    <p>Can the kernel represent geometry that doesn't exist physically? A face with no thickness, an edge shared by three faces, a wireframe floating in space. Parasolid supports this natively (Convergent Modeling). ACIS has partial support. This matters for simulation prep — FEA meshers often need non-manifold geometry that a strict solid modeler would reject.</p>
  </div>
</div>

<h3 style="margin-top:2rem">Where the Names Come From</h3>
<div style="font-size:0.8rem;color:var(--text-secondary);line-height:1.8;max-width:65ch">
  <p><strong style="color:var(--text-primary)">Parasolid</strong> — "Parametric Solid." Developed by Shape Data Ltd (Cambridge, UK) in 1985 as the parametric successor to Romulus. The name signals the key innovation: geometry that remembers its construction history and can be edited by changing parameters.</p>
  <p><strong style="color:var(--text-primary)">ACIS</strong> — Named after the Greek myth of Acis and Galatea (from Ovid's <em>Metamorphoses</em>). Unofficially also parsed as "Alan, Charles, Ian's System" — Alan Grayer, Charles Lang, and Ian Braid, the Cambridge researchers who built it. Created at Spatial Technology in 1989. The <code>.SAT</code> file format stands for "Save As Text."</p>
  <p><strong style="color:var(--text-primary)">CGM</strong> — "Convergence Geometric Modeler." Built by Dassault in the late 1990s for CATIA V5 when they decided ACIS wasn't meeting their aerospace requirements. "Convergence" refers to unifying wireframe, surface, and solid modeling in one engine.</p>
  <p><strong style="color:var(--text-primary)">Granite</strong> — PTC's internal name. Built for Pro/ENGINEER (1988). Never given a marketing push because PTC never licensed it externally — it's a captive kernel.</p>
  <p><strong style="color:var(--text-primary)">Open CASCADE</strong> — Derived from Matra Datavision's CAS.CADE system (Computer Aided Software for Computer Aided Design and Engineering). Open-sourced in 1999. The most important open-source B-rep kernel, now the foundation of FreeCAD, CadQuery, and browser-based CAD.</p>
  <p><strong style="color:var(--text-primary)">ShapeManager</strong> — Autodesk's name for their 2001 fork of ACIS 7.0. Descriptive name: it manages shapes. Autodesk forked it to stop paying per-seat licensing fees to Spatial. Every AutoCAD, Inventor, and Fusion 360 seat runs on it.</p>
  <p><strong style="color:var(--text-primary)">Spatial</strong> — Dassault subsidiary (acquired 2000) that licenses ACIS and CGM to the world. Based in Colorado. When someone says "we license ACIS," the contract is with Spatial, not Dassault directly. Also sells 3D InterOp (file translation between kernels).</p>
  <p><strong style="color:var(--text-primary)">openNURBS / Rhino</strong> — McNeel's own NURBS kernel, built from scratch in Seattle. Not ACIS, not Parasolid, not OCCT — completely independent. NURBS-first (surfaces are the primary citizen, not solids), which is why Rhino dominates freeform design. The <em>openNURBS</em> toolkit is open-source for reading/writing .3dm files, but the actual modeling engine is proprietary.</p>
</div>

<h2>Why This Matters for CAM</h2>

<div class="paradigm-card" style="border-color:var(--accent-dim);border-left:3px solid var(--accent);margin-bottom:1.5rem">
  <h3 style="color:var(--accent)">Mastercam runs on Parasolid</h3>
  <p>The same kernel as SolidWorks, NX, Solid Edge, Onshape, and Shapr3D. That's why <code>.x_t</code> (Parasolid) files round-trip cleanly between these tools — it's the same mathematical representation on both ends. No translation, no information loss.</p>
</div>

<h3>STEP &gt; STL — Always</h3>
<p>A STEP file carries exact B-rep geometry: NURBS surfaces with true normals at every point. An STL file is a bag of triangles — an approximation. Your 5-axis finish pass computes cutter-contact points against those surfaces. Exact NURBS = glass-smooth. Triangulated mesh = facets.</p>

<h3 style="margin-top:1.5rem">The SDF-to-CAM Gap</h3>
<p>If your part is designed in nTopology (implicit lattices for additive manufacturing), it must be meshed to STL before any CAM system can touch it. The mathematical function that defines the lattice is <em>destroyed</em> in the conversion. This is the biggest unsolved interoperability problem in manufacturing today.</p>

<h3 style="margin-top:1.5rem">What the Kernel Sees</h3>
<p>Your CAM software doesn't see your model the way you do. It sees offset surfaces (for cutter compensation), containment boundaries (for gouge detection), and surface normal vectors (for 5-axis tool orientation). All computed by the kernel, thousands of times per toolpath segment.</p>

<h3 style="margin-top:1.5rem">Why Some CAM Vendors Build Their Own Kernels</h3>
<p>Most CAM software licenses Parasolid — Mastercam, GibbsCAM, Esprit, Edgecam, PowerMill all pay Siemens per seat. But a few built their own:</p>
<div class="failure-grid" style="grid-template-columns:1fr">
  <div class="failure-card">
    <h3>hyperMILL → hyperCAD-S (OPEN MIND, Germany)</h3>
    <p>Not a full kernel like Parasolid. It's a lightweight CAD viewer/editor purpose-built for CAM workflows: stock model management, electrode design, 5-axis setup. CAM doesn't need 90% of what Parasolid offers (parametric history, assembly constraints). So OPEN MIND built exactly what CAM needs and nothing more. Also eliminates per-seat Parasolid licensing costs. Escape hatch: hyperMILL also runs as a plugin inside SolidWorks or Inventor, using <em>their</em> kernel instead.</p>
  </div>
  <div class="failure-card">
    <h3>Tebis → Own kernel (Germany)</h3>
    <p>Tebis built their own geometric engine decades ago, specialized for die/mold and aerospace surfacing. Their strength is 5-axis simultaneous machining of complex freeform surfaces — toolpath quality on deep cavities and thin walls. The kernel is optimized for the specific surface interrogation that mold CAM demands (draft analysis, undercut detection, parting surface generation).</p>
  </div>
</div>
<p style="font-size:0.8rem;color:var(--text-secondary);margin-top:0.75rem">The trend is toward Parasolid — it's become the "just license this" default for new CAM entrants. But the holdouts built their own for a reason: a kernel tuned for CAM-specific geometry operations can outperform a general-purpose kernel on the operations that matter most.</p>

<h3 style="margin-top:1.5rem">What's Next: Convergent Modeling</h3>
<p>Siemens is betting on mixing mesh and B-rep in the same model — scan data, generative design output, and precision features coexisting. They call it <em>Convergent Modeling</em>. If it works, the SDF-to-CAM gap closes. Your CAM kernel would handle both representations natively.</p>

K.mod.sdf = {
  inited: false,
  sdfCtx: null, ctrCtx: null,
  W: 120, H: 120, CW: 280, CH: 280,
  buf: null, cbuf: null,

  init() {
    if (this.inited) return;
    this.inited = true;
    const sc = document.getElementById('sdfCanvas');
    const cc = document.getElementById('contourCanvas');
    if (!sc || !cc) return;
    this.sdfCtx = sc.getContext('2d');
    this.ctrCtx = cc.getContext('2d');
    this.buf = new Uint8ClampedArray(this.W * this.H * 4);
    this.render();
  },

  // SDF primitives
  sdCircle(x, y, r) { return Math.hypot(x, y) - r; },
  sdBox(x, y, bx, by) {
    const dx = Math.abs(x) - bx, dy = Math.abs(y) - by;
    return Math.hypot(Math.max(dx, 0), Math.max(dy, 0)) + Math.min(Math.max(dx, dy), 0);
  },
  smin(a, b, k) {
    if (k < 0.001) return Math.min(a, b);
    k *= 4; const h = Math.max(k - Math.abs(a - b), 0);
    return Math.min(a, b) - h * h * 0.25 / k;
  },

  scene(x, y) {
    const { blend, offset } = K.state.sdf;
    const box = this.sdBox(x, y, 0.35, 0.22);
    const c1 = this.sdCircle(x - 0.25 - offset, y + 0.1, 0.18);
    const c2 = this.sdCircle(x + 0.3, y - 0.15 + offset, 0.14);
    return this.smin(this.smin(box, c1, blend), c2, blend);
  },

  turbo(t) {
    t = Math.max(0, Math.min(1, t));
    const t2=t*t, t3=t2*t, t4=t3*t, t5=t4*t;
    return [
      Math.max(0, Math.min(1, 0.13572+4.61539*t-42.6603*t2+132.131*t3-152.942*t4+59.2864*t5)),
      Math.max(0, Math.min(1, 0.09140+2.19419*t+4.84297*t2-14.185*t3+4.27730*t4+2.82957*t5)),
      Math.max(0, Math.min(1, 0.10667+12.6419*t-60.582*t2+110.363*t3-89.903*t4+27.3482*t5))
    ];
  },

  render() {
    const { W, H, buf, sdfCtx, ctrCtx, CW, CH } = this;
    if (!sdfCtx) return;
    // Heatmap
    for (let py = 0; py < H; py++) {
      for (let px = 0; px < W; px++) {
        const x = (px / W - 0.5) * 2, y = (0.5 - py / H) * 2;
        const d = this.scene(x, y);
        const t = Math.max(0, Math.min(1, d / 0.6 + 0.5));
        const [r, g, b] = this.turbo(t);
        const i = (py * W + px) * 4;
        buf[i] = r * 255 | 0; buf[i+1] = g * 255 | 0; buf[i+2] = b * 255 | 0; buf[i+3] = 255;
      }
    }
    sdfCtx.putImageData(new ImageData(buf, W, H), 0, 0);

    // Contour canvas: solid fill + zero isoline
    ctrCtx.fillStyle = '#111318';
    ctrCtx.fillRect(0, 0, CW, CH);
    // Fill inside
    const img2 = ctrCtx.createImageData(CW, CH);
    const d2 = img2.data;
    for (let py = 0; py < CH; py++) {
      for (let px = 0; px < CW; px++) {
        const x = (px / CW - 0.5) * 2, y = (0.5 - py / CH) * 2;
        const dist = this.scene(x, y);
        const i = (py * CW + px) * 4;
        if (dist < 0) { d2[i]=30; d2[i+1]=60; d2[i+2]=80; d2[i+3]=255; }
        else { d2[i]=17; d2[i+1]=19; d2[i+2]=24; d2[i+3]=255; }
      }
    }
    ctrCtx.putImageData(img2, 0, 0);
    // Draw zero isoline
    const G = 80;
    ctrCtx.strokeStyle = '#2eb8a0';
    ctrCtx.lineWidth = 2;
    ctrCtx.beginPath();
    for (let gy = 0; gy < G - 1; gy++) {
      for (let gx = 0; gx < G - 1; gx++) {
        const x0 = (gx/G-0.5)*2, y0=(0.5-gy/G)*2;
        const x1 = ((gx+1)/G-0.5)*2, y1=(0.5-(gy+1)/G)*2;
        const d00=this.scene(x0,y0), d10=this.scene(x1,y0);
        const d01=this.scene(x0,y1), d11=this.scene(x1,y1);
        const tc = (wx, wy) => [(wx*0.5+0.5)*CW, (0.5-wy*0.5)*CH];
        const lerp = (a, b) => a / (a - b);
        const pts = [];
        if ((d00<0)!==(d10<0)) { const f=lerp(d00,d10); pts.push(tc(x0+f*(x1-x0),y0)); }
        if ((d10<0)!==(d11<0)) { const f=lerp(d10,d11); pts.push(tc(x1,y0+f*(y1-y0))); }
        if ((d01<0)!==(d11<0)) { const f=lerp(d01,d11); pts.push(tc(x0+f*(x1-x0),y1)); }
        if ((d00<0)!==(d01<0)) { const f=lerp(d00,d01); pts.push(tc(x0,y0+f*(y1-y0))); }
        if (pts.length >= 2) { ctrCtx.moveTo(pts[0][0],pts[0][1]); ctrCtx.lineTo(pts[1][0],pts[1][1]); }
      }
    }
    ctrCtx.stroke();
    // Draw some edge labels
    ctrCtx.fillStyle = 'rgba(46,184,160,0.7)';
    ctrCtx.font = '10px IBM Plex Mono';
    ctrCtx.fillText('face', 100, 130);
    ctrCtx.fillText('edge', 170, 90);
    ctrCtx.fillText('vertex', 60, 180);
  }
};

K.mod.graph = {
  init() {
    const svg = document.getElementById('graphSvg');
    const edgeG = document.getElementById('graphEdges');
    const nodeG = document.getElementById('graphNodes');
    const ns = 'http://www.w3.org/2000/svg';

    // Draw edges
    EDGES.forEach(([fromId, toId, type]) => {
      const from = NODES.find(n => n.id === fromId);
      const to = NODES.find(n => n.id === toId);
      if (!from || !to) return;
      const x1=from.x, y1=from.y, x2=to.x, y2=to.y;
      const cpx=(x1+x2)/2, cpy=y1+(y2-y1)*0.3;
      const path = document.createElementNS(ns, 'path');
      path.setAttribute('d', `M${x1} ${y1} Q${cpx} ${cpy} ${x2} ${y2}`);
      path.setAttribute('class', 'edge');
      path.dataset.from = fromId;
      path.dataset.to = toId;
      if (type === 'ancestor' || type === 'fork') path.setAttribute('stroke-dasharray', '4 3');
      path.setAttribute('marker-end', type === 'owns' ? 'url(#arrowOwn)' : 'url(#arrowUse)');
      edgeG.appendChild(path);
    });

    // Draw nodes
    NODES.forEach(n => {

      // Invisible touch target
      const hit = document.createElementNS(ns, 'circle');
      hit.setAttribute('cx', n.x); hit.setAttribute('cy', n.y);
      hit.setAttribute('r', Math.max(n.r + 8, 22));
      hit.setAttribute('fill', 'transparent');
      g.appendChild(hit);
      // Visible circle
      const c = document.createElementNS(ns, 'circle');
      c.setAttribute('cx', n.x); c.setAttribute('cy', n.y);
      c.setAttribute('r', n.r);
      c.setAttribute('fill', n.color);
      c.setAttribute('class', 'node-circle');
      c.setAttribute('pointer-events', 'none');
      c.setAttribute('opacity', '0.85');
      g.appendChild(c);

      const t = document.createElementNS(ns, 'text');
      t.setAttribute('x', n.x); t.setAttribute('y', n.y - (n.sub ? 3 : 0));
      t.setAttribute('text-anchor', 'middle');

      t.setAttribute('font-size', n.r > 24 ? '11' : n.r > 16 ? '9' : '7');

      else this.hidePanel();

    // Pan/zoom via viewBox
    let vb = { x: 0, y: 0, w: 1000, h: 720 };
    const applyVB = () => svg.setAttribute('viewBox', `${vb.x} ${vb.y} ${vb.w} ${vb.h}`);
    const clampZoom = () => { vb.w = Math.max(300, Math.min(2000, vb.w)); vb.h = vb.w * (720 / 1000); };

    // Single-finger pan
    let panning = false, panStart = {};
    svg.addEventListener('pointerdown', e => {
      if (e.target.closest('[data-node-id]')) return;
      panning = true; panStart = { px: e.clientX, py: e.clientY, vx: vb.x, vy: vb.y };
      svg.setPointerCapture(e.pointerId);
    });
    svg.addEventListener('pointermove', e => {
      if (!panning) return;
      const s = vb.w / svg.clientWidth;
      vb.x = panStart.vx - (e.clientX - panStart.px) * s;
      vb.y = panStart.vy - (e.clientY - panStart.py) * s;
      applyVB();
    });
    svg.addEventListener('pointerup', () => { panning = false; });

    // Mouse wheel zoom
    svg.addEventListener('wheel', e => {
      e.preventDefault();
      const f = e.deltaY > 0 ? 1.08 : 0.92;
      const rect = svg.getBoundingClientRect();
      const mx = (e.clientX - rect.left) / rect.width * vb.w + vb.x;
      const my = (e.clientY - rect.top) / rect.height * vb.h + vb.y;
      vb.w *= f; vb.h *= f; clampZoom();
      vb.x = mx - (e.clientX - rect.left) / rect.width * vb.w;
      vb.y = my - (e.clientY - rect.top) / rect.height * vb.h;
      applyVB();
    }, { passive: false });

    // Pinch-to-zoom + two-finger pan (mobile)
    let lastPinch = null;
    const pinchData = (ts) => ({
      cx: (ts[0].clientX + ts[1].clientX) / 2,
      cy: (ts[0].clientY + ts[1].clientY) / 2,
      dist: Math.hypot(ts[1].clientX - ts[0].clientX, ts[1].clientY - ts[0].clientY)
    });
    svg.addEventListener('touchstart', e => {
      if (e.touches.length === 2) { e.preventDefault(); panning = false; lastPinch = pinchData(e.touches); }
    }, { passive: false });
    svg.addEventListener('touchmove', e => {
      if (e.touches.length === 2 && lastPinch) {
        e.preventDefault();
        const cur = pinchData(e.touches);
        const rect = svg.getBoundingClientRect();
        const scale = lastPinch.dist / cur.dist;
        // Zoom around pinch midpoint
        const mx = (cur.cx - rect.left) / rect.width * vb.w + vb.x;
        const my = (cur.cy - rect.top) / rect.height * vb.h + vb.y;
        vb.w *= scale; vb.h *= scale; clampZoom();
        vb.x = mx - (cur.cx - rect.left) / rect.width * vb.w;
        vb.y = my - (cur.cy - rect.top) / rect.height * vb.h;
        // Pan from midpoint movement
        const s = vb.w / rect.width;
        vb.x -= (cur.cx - lastPinch.cx) * s;
        vb.y -= (cur.cy - lastPinch.cy) * s;
        applyVB();
        lastPinch = cur;
      }
    }, { passive: false });
    svg.addEventListener('touchend', () => { lastPinch = null; });
  },

  showPanel(id) {
    const info = NODE_INFO[id];
    if (!info) return;
    document.getElementById('panelTitle').textContent = info.title;
    document.getElementById('panelMeta').textContent = info.meta;
    document.getElementById('panelDesc').textContent = info.desc;
    const list = document.getElementById('panelList');
    list.innerHTML = (info.connections || []).map(c => `<li>${c}</li>`).join('');
    document.getElementById('graph-panel').classList.add('open');
    // Highlight edges
    document.querySelectorAll('#graphEdges .edge').forEach(e => {
      e.classList.toggle('highlighted', e.dataset.from === id || e.dataset.to === id);
    });
  },
  hidePanel() {
    document.getElementById('graph-panel').classList.remove('open');
    document.querySelectorAll('#graphEdges .edge').forEach(e => e.classList.remove('highlighted'));
  }

    this.renderBody();
    document.querySelectorAll('#kernelTable th').forEach(th => {

        const col = th.dataset.col;
        if (K.state.sort.col === col) K.state.sort.dir *= -1;

        this.renderBody();
        this.updateArrows();
      });
    });
  },

  renderBody() {
    const { col, dir } = K.state.sort;
    const sorted = [...TABLE_DATA].sort((a, b) => {
      const av = a[col], bv = b[col];
      if (typeof av === 'number') return (av - bv) * dir;
      return String(av).localeCompare(String(bv)) * dir;
    });
    const stars = n => '★'.repeat(n) + '☆'.repeat(5-n);
    const tb = (t) => t.includes('B-rep') ? 'badge-brep' : t.includes('SDF') || t.includes('rep') || t.includes('Implicit') ? 'badge-sdf' : 'badge-brep';
    const ob = (o) => o === 'Yes' ? 'badge-yes' : o === 'No' ? 'badge-no' : 'badge-api';
    document.getElementById('kernelBody').innerHTML = sorted.map(k => `<tr>
      <td style="color:var(--text-primary);font-weight:600">${k.name}</td>
      <td><span class="badge ${tb(k.type)}">${k.type}</span></td>
      <td>${k.lang}</td>
      <td><span class="${ob(k.open)}">${k.open}</span></td>
      <td class="maturity">${stars(k.maturity)}</td>
      <td>${k.strength}</td>
    </tr>`).join('');
  },

  updateArrows() {
    const { col, dir } = K.state.sort;
    document.querySelectorAll('#kernelTable th').forEach(th => {
      if (th.dataset.col === col) th.setAttribute('aria-sort', dir === 1 ? 'ascending' : 'descending');

          e.target.classList.add('in-view');
          const idx = [...sections].indexOf(e.target);
          fill.style.width = `${((idx + 1) / sections.length) * 100}%`;
          links.forEach((l, i) => l.classList.toggle('active', i === idx));
          // Lazy-init SDF
          if (e.target.id === 's2') K.mod.sdf.init();
        } else {
          e.target.classList.remove('in-view');
        }
      });
    }, { threshold: 0.15 });
    sections.forEach(s => obs.observe(s));
  }
};
